<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ESP32 ↔ WiFi router | Unencrypted WiFi association; credentials stored in NVS |
| ESP32 ↔ Aggregator host | UDP frames over local network; no encryption |
| Host PC ↔ ESP32 (serial) | Physical USB-UART; provisioning writes credentials to NVS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | NVS config storage | accept | NVS is internal flash; physical access required to extract. No PII beyond WiFi credentials. |
| T-01-02 | Denial of Service | UDP stream sender | mitigate | Implement ENOMEM backoff in stream_sender.c (100ms cooldown) per reference pattern. |
| T-01-03 | Tampering | Serial provisioning | accept | Physical USB access required; out of v1 threat model. |
| T-01-04 | Spoofing | CSI frame magic | mitigate | Validate magic `0xC5110001` on receive side (Phase 2 aggregator). |
</threat_model>
