<task type="auto">
  <name>Task 1: Implement SlidingWindow buffer</name>
  <files>processor/window.py, tests/test_window.py</files>
  <action>
    1. Create `processor/window.py` with class `SlidingWindow`:
       - `__init__(self, n_subcarriers: int = 64, window_size: int = 200, step_size: int = 100)`:
         - window_size = 200 frames (4s @ 50 Hz), step_size = 100 frames (2s overlap per D-17)
         - Internal buffer: numpy array shape `(window_size, n_subcarriers)` for amplitudes
         - Counter tracking how many frames received since last emission
       - `push(frame: CSIFrame) -> np.ndarray | None`:
         - Extract amplitude array from frame (length n_subcarriers)
         - Append to internal buffer (circular/roll strategy for efficiency)
         - Increment counter
         - If counter >= step_size AND buffer has >= window_size frames:
           - Emit a copy of the current window as `(window_size, n_subcarriers)` numpy array
           - Reset counter to 0, slide buffer by step_size (keep last window_size - step_size frames)
         - Return None if no window ready yet
       - `is_full() -> bool`: Returns True if buffer has >= window_size frames
       - `reset() -> None`: Clear buffer and counter
    2. Handle edge case: frame.amplitudes may be list → convert to np.array
    3. Create `tests/test_window.py`:
       - Push 200 frames → expect 1 window returned, shape (200, 64)
       - Push 300 frames → expect 1 window at frame 200, then counter at 100
       - Push 400 frames → expect 2 windows (at 200 and 300)
       - Verify window content matches input amplitudes (not zeros or stale data)
       - Test reset() clears state
  </action>
  <verify>
    <automated>pytest tests/test_window.py -x</automated>
  </verify>
  <done>
    - SlidingWindow emits windows of shape (200, 64) after 200 frames
    - Second window emitted after 300 frames total (200 + 100 step)
    - Buffer correctly slides without data corruption
  </done>

<task type="auto">
  <name>Task 2: Implement feature extraction</name>
  <files>processor/features.py, tests/test_features.py</files>
  <action>
    1. Create `processor/features.py` with function:
       - `extract_features(amplitude_window: np.ndarray, phase_window: np.ndarray | None = None, sample_rate: float = 50.0) -> dict[str, np.ndarray | float]`
       - Input `amplitude_window`: shape `(200, 64)` — 200 time steps, 64 subcarriers
       - Input `phase_window`: optional, shape `(200, 64)` — unwrapped and detrended phases
       - Compute features per D-16:
         - `mean_amp`: shape `(64,)` — mean amplitude per subcarrier across time axis
         - `var_amp`: shape `(64,)` — variance per subcarrier across time axis
         - `motion_energy`: float — band power 0.5-3 Hz. Compute via FFT:
           - For each subcarrier, take FFT of amplitude time series (200 samples)
           - Sum power (magnitude squared) in bins corresponding to 0.5-3 Hz
           - Average across all 64 subcarriers, return scalar
         - `breathing_band`: float — band power 0.1-0.5 Hz. Same FFT method, sum 0.1-0.5 Hz bins.
       - Return dict exactly as specified in D-16:
         ```python
         {
           "mean_amp": np.ndarray,      # shape (64,)
           "var_amp": np.ndarray,       # shape (64,)
           "motion_energy": float,
           "breathing_band": float,
         }
         ```
    2. FFT computation details:
       - Use `np.fft.rfft` on each subcarrier's time series (axis=0)
       - Frequency resolution = sample_rate / n_samples = 50/200 = 0.25 Hz
       - Bin indices: 0.5 Hz = bin 2, 3 Hz = bin 12, 0.1 Hz = bin 0 (use bin 1 for clean separation), 0.5 Hz = bin 2
       - Sum `np.abs(fft)**2` over the bin range
    3. If `phase_window` is provided (for future presence detection breathing sensitivity), also compute:
       - `phase_variance`: float — mean variance of detrended phase across subcarriers
       - Add to return dict as optional key
    4. Create `tests/test_features.py`:
       - Constant amplitude window (all 1.0) → mean_amp ≈ 1.0, var_amp ≈ 0, motion_energy ≈ 0, breathing_band ≈ 0
       - Synthetic sine wave at 1 Hz in amplitude → motion_energy > 0, breathing_band ≈ 0
       - Synthetic sine wave at 0.3 Hz in amplitude → breathing_band > 0, motion_energy ≈ 0
       - Verify output dict has correct keys and array shapes
       - Verify feature count: len(mean_amp) + len(var_amp) + 1 + 1 = 130 scalar values
  </action>

    - motion_energy and breathing_band are scalar floats
    - Total feature count = 130 scalar values
    - Band separation works: 1 Hz signal → motion_energy > 0, breathing_band ≈ 0
  </done>
