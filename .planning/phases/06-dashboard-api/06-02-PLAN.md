## UI Decisions (Locked)

- **D-45:** Vanilla HTML/JS + Canvas 2D, zero frontend dependencies. Single `index.html` with inline CSS and `<script>` tags.
- **D-47:** 5-panel grid layout:
  - Panel 1 (top-left, largest): CSI amplitude heatmap
  - Panel 2 (top-right): Presence status indicator (green=empty, red=occupied)
  - Panel 3 (bottom-left): Activity label with confidence bar
  - Panel 4 (bottom-right): Alert log panel (intrusion events with timestamp + node ID)
  - Panel 5 (sidebar/header): Node health (FPS per node, stale/connected status)
- **D-48:** Heatmap: subcarrier (Y-axis, up to 52 rows) vs time (X-axis, last ~20s / 200 frames). Color scale: blue (low) → green → yellow → red (high). Canvas 2D `fillRect` per cell.
- **D-46:** WebSocket at fixed 2Hz interval (every 500ms). JSON bundle with all panels.

    **JavaScript requirements (inline in `<script>`):**
    - `const WS_URL = \`ws://${window.location.host}/ws\`;`
    - `const RECONNECT_BASE = 1000; const RECONNECT_MAX = 30000;`
    - State object: `{ ws: null, reconnectAttempts: 0, reconnectTimer: null }`
    - `connect()` function:
      - Create `new WebSocket(WS_URL)`
      - `onopen`: reset `reconnectAttempts`, update connection-status to "connected" (green)
      - `onmessage`: parse JSON, call `updatePanels(data)`
      - `onclose`: update connection-status to "disconnected", schedule reconnect with `setTimeout(connect, Math.min(RECONNECT_BASE * 2 ** reconnectAttempts, RECONNECT_MAX))`, increment `reconnectAttempts`
      - `onerror`: log to console
    - `updatePanels(data)` function:
      - **Heatmap**: for each node in `data.heatmap`, pick the first node (or render all in tabs if multiple). Use Canvas 2D:
        - `const ctx = canvas.getContext('2d');`
        - `const rows = heatmapData.length; const cols = heatmapData[0]?.length || 0;`
        - `const cellW = canvas.width / cols; const cellH = canvas.height / rows;`
        - For each cell: `ctx.fillStyle = getColor(value); ctx.fillRect(col * cellW, row * cellH, cellW, cellH);`
      - `getColor(value)` function: normalize value to 0-1 using min/max across all visible data (or use fixed range 0-0.5). Map to blue→green→yellow→red:
        - `0.0 - 0.33`: interpolate between `#0000ff` and `#00ff00`
        - `0.33 - 0.66`: interpolate between `#00ff00` and `#ffff00`
        - `0.66 - 1.0`: interpolate between `#ffff00` and `#ff0000`
        - Return `rgb(r, g, b)` string
      - **Presence**: for each node in `data.presence`, create a circle div with class based on status. Show text label below circle.
      - **Activity**: for the first active node in `data.activity`, show label text and confidence bar. If no activity, show "No activity detected".
      - **Alerts**: clear container, append up to 10 most recent alerts (from `data.alerts`). Format timestamp to locale time string. Show node ID badge, status text, and type badge.
      - **Node Health**: build table rows from `data.node_health`. Show FPS as integer. Show "ONLINE" in green or "STALE" in red based on `stale` boolean.
    - `resizeCanvas()` function: set `canvas.width = canvas.clientWidth * window.devicePixelRatio; canvas.height = canvas.clientHeight * window.devicePixelRatio; ctx.scale(window.devicePixelRatio, window.devicePixelRatio);` then trigger redraw. Call on window resize.
    - Call `connect()` on DOMContentLoaded.
    - Also call `fetch('/status').then(r => r.json()).then(updatePanels)` on initial load for immediate data before WebSocket connects.

    **Do NOT use external libraries** (no CDN links, no React, no Vue, no D3, no Chart.js).
    **Do NOT use ES6 modules** — all code in single inline script for simplicity.
    **Do NOT split into separate .css or .js files** — everything inline in index.html.
